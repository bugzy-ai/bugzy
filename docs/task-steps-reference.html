Step Types Explained

  • Library Steps: Reusable across multiple tasks (defined in STEP_LIBRARY)
  • Inline Steps: Task-specific content defined directly in task file
  • Every task needs: Overview (inline) + Security Notice (library) + Arguments (inline)
  • Pattern: { inline: true, title: '...', content: '...' }
